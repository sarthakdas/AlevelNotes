Applications Generation

Software may be grouped into separate categories:

Systems Software

System Software is the software needed to run the computer's hardware and application program.

  • Operating Systems
  • Library Programs
  • Utility Programs
  • Translators

Application Software

  • Off-the-Shelf
  • Custom Written
  • Proprietary
  • Open Source

Systems Software

Operating Systems

It has many functions such as resource management, such as communicating with all the components and hardware

Provision of UI to enable the user to preform tasks such as running application software

Utility Programs

It is a system software designed ti optimise the performance of the computer or preform tasks such as backing up files, encryption and providing a firewall.

  • Disk Defragmentation

    A disk defragmenter is a program that will reorganise a magnetic hard-disk so that files which have been split up will be recombined, making it easier to read.

  • Automatic Backup

    This allows the user to specify where they want the back up to be stored, what they want to back up and how to back them up and when to run the backups.

  • Automatic Updating

    An automatic update utility makes sure that any software installed is up-to-date. it regularly checks the internet for updates and these will then be downloaded.

  • Virus Checker

    A virus checker scans the hard-disk for any files that can be viruses and deletes them

Compression Software

Several utility programs are included to copy, move, and delete files. Some other software is included for compression this makes it easier to transmit files across networks.

Application Software

These software can be categorised as general-purpose, special-purpose or custom-written.

General-purpose software such as word-processor, spreadsheets and powerpoint.

Special-purpose software preforms single specific task, such as payroll management and booking system.

Open Source vs Proprietary

Open Source

It is software that has no charge, and that the source code is openly available so anyone can modify it. Any software created from open source software must be "open"

Proprietary

It is sold with restrictions on how the software can be used, for example the license may specify only one user.

Programming Language Translators

Assembler

It is a low level language, with each instruction being equivalent to one machine code instruction. The machine code instructions can only execute on that processor, so each processor has its own instruction set.

Before an assembly code and be run it must be translated into assembly code, which is done by the assembler. The input to the assembler is called the source code and the output is called the object code.

Complier

A compiler is a program that translates a high level language into machine code. The input is scanned multiple times each time preforming different checks and building tables of information needed to produce the final object code.

Different hardware platforms require different compilers. Object code can be saved and run again without the presence of a compiler.

Interpreter

An interpreter is a type of translator. It looks at each individual line of code, analyses it, and if there are no errors it translates that line into machine code and runs it.

Stages of Compilation

There are 3 stages of compilation lexical analysis, syntax analysis, code generation and optimisation.

Lexical Analysis

  • Superfluous spaces are removed
  • All comments are removed
  • Simple error checking
  • Keywords, variables, constants and identifiers are replaced by tokens (symbol table)

The symbol table plays a central role in compilation. It contains every keyword in the program. It stores:

  • The keyword
  • The kind of item (variable, array, procedure)
  • The type (integer, real, float)
  • The run-time address
  • Pointer to accessing information (bounds in an array)

Since the lexical analyser spends a long time looking up the symbol table. The most common way of organising it is using a hash table, where each keyword is hashed to produce an array subscript.

Syntax Analysis

It is the process of determining if the sentence makes sense in the language.

Parsing is the task of applying a set of rules to check if it makes sense. Stacks will be used to check if brackets are correctly paired. All expressions will be converted to reverse polish notation from which machine code can more easily be generated.

Semantics define the meaning of the code rather then the grammar. Such as check if an undeclared variable has been used.

Code Generation and Optimisation

Code optimisation attempt to reduce the execution time of the object program, by eliminating redundant instructions.

However this can increase code compilation time, and can lead to unexpected results.

Links and Loaders

Once code is complied any subroutines must be added, such as random number generator. It is the job of the linker to put the appropriate machine addresses in.

Use of libraries

These are pre-complied programs that provide extra functions that can be used from the program.

A couple of advantages are that the code used is error free and it saves the user from re-inventing the wheel.
