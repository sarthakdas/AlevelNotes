Compression, Encryption and Hashing

Compression

File compression techniques are used to reduce the storage space on files. This is important to store more files in the sane device and to send files quicker of the internet which has bandwidth limitations.

Compression can be either lossy, where unnecessary information is removed from the original file or lossless, where it retains all the information to reproduce the file exactly.

Lossy Compression

This works by removing non-essential information. A file that has been compressed (generally images) appear blocky and have compression artefacts, but still looks more or less the same

This works similar with music and video compression. MP3 files use lossy compression to remove frequencies too high for most of us to hear.

Lossless Compression

Lossless compression works by recording patterns in the data. Using the patterns and a set of instructions the computer can reverse the procedure and and reassemble the image. This is mostly used in situations where a single character lost would effect the meaning and cause errors. They still tend to be large but still smaller then the original.

Run Length Encoding (RLE)

Run length encoding is where is where rather then encoding every pixel un a sequence it records its value and the number of times it repeats.

This reduces the size of the file and does not reduce the image quality.

Dictionary-based Compression Techniques

Imagine sending a dictionary along side a coded message using the page number and the position of the word on the page. This would be more efficient and requires less data.

Dictionary based compression works in a similar way. The compression algorithm searches through the text to find suitable entries in its own dictionary and translates the message

Encryption

Encryption is is the transformation of data from one form to another to prevent a third party from being able to understand it.

Symmetric (Private Key) Encryption

It is when the same key is used for encryption and decryption. This means that the key must also be transferred to the same destination. The key can be intercepted therefore can make messaging insecure.

Asymmetric (Public Key) Encryption

This used 2 separate keys. The public key is made public so that others wishing to send you data can use this to encrypt the data. The private key is known only by you and can be used to decrypt the data. Each message is also signed to authenticate the sender to prevent a third party from mimicking the real sender. This type of encryption can only be used for a certain amount of data.

Hashing

This provides mapping between an arbitrary length input and a usually fixed length for smaller output. Unlike encryption techniques, this is one way. Useful to encrypt pins and passwords.

Hash Functions

A hash total is a mathematical value calculated from unencrypted message data. This value is referred to as the checksum. The process is irreversible. The slightest change will result in a totally different hash. Multiple messages my result in the same hash.

Digital Signatures

This hash value is like a signature. The sender uses their private key to encrypt the message. This becomes the digital signature. This is then attached to the message and then encrypted again using the recipients public key.

The recipient decrypts it with their private key. and then decrypts the digital signature using the senders public key. The hash total is then reproduced based on the message to confirm that no parts where changed on transition.

Digital Certificates

This is issued by certificate authorities to verify the trustworthiness of a mesage sender or website. This allows the user to use the Public Key Infrastructure to authenticate signitures and certificates. This operates on the TCP/IP protocol stack.
