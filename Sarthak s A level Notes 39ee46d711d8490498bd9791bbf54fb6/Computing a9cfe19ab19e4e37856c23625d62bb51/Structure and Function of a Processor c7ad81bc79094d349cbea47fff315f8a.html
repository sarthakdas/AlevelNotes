Structure and Function of a Processor

The Central Processing Unit (CPU)

This contains various other components that help it carry out its task. These components include:

  • Control Unit
  • Buses
  • Arithmetic-Logic Unit (ALU)
  • Dedicated Registers

Control Unit

This component controls and coordinates the activities, directing the flow of data. It decodes instructions and fetches addresses from memory and manages its execution .

Buses

A bus is a set of parallel wires connecting two or more components. The processor is connected by 3 buses.

When it requires a address it uses the address bus. It is then returned to the CPU using the data bus. Control signals are sent along the control bus. These buses are known as the system buses.

Data and control signals can travel in both directions while addresses can only travel in one direction.

  • Data Bus

    Consisting of either 8, 16, 32, 64 separate lines, provides a bi-directional system for moving data between components.

  • Address Bus

    Memory is divided into units called words. A word is a fixed size of digits typically 16,32,64 bits. Different types of processors have different word sizes.

    Each word has its own address. Address bus transmits the memory address so data can be retrieved and sent

  • Control Bus

    It is bi-directional. Its purpose is to transmit commands, timings, and specific information

    Control lines include:

    • Bus Request - requesting use of the line
    • Bus Grant - indicates permission is granted
    • Memory Write - causes data to be written
    • Memory Read - causes data to be read
    • Interrupt Request - request to access CPU
    • Clock - used to synchronise operations

Arithmetic-Logic Unit (ALU)

This preforms logical operations on the data and can do addition, subtraction, multiplication and division on fixed or floating point numbers. It can also carry out boolean logic operations comparing values such as and, or, not, xor.

Registers

Registers operate at high speeds and complex computers can have many of these. The main one is the accumulator where all calculations take place.

There are several other special purpose registers:

Program counter

Holds the address of the next instruction to be executed, copied from the current instruction register.

Current Instruction Register (CIR)

Holds the current instruction being executed divided into operand and opcode

Memory Address Register (MAR)

Holds the address of the memory location of data that needs to be fetched or written

Memory Data Register (MDR)

Temporarily holds data that is read or written to memory, also known as memory buffer register

The Fetch-Decode Execute Cycle

The sequence of operations involved in executing an instruction can be divided into 3 phases- fetch, execute, decode.

  • Fetch
    • The address of the next instruction is copied from the program counter (PC) to the memory address register (MAR)
    • It is then copied to the memory data register (MDR) and the program counter (PC) is incremented so it holds the next instruction
    • Contents of the memory data register (MDR) is copied to the current instruction register (CIR)
  • Decode
    • The instruction is held in the CIR is decoded split into opcode and operand.

      Opcode - Type of instruction and what hardware required

      Operand - Holds the address of the data or actual data or data to be operated on

  • Execute
    • The appropriate instructions are carried out by the operand

Processor performance

The main factors affecting processor performance are:

  • Clock Speed
  • Number of cores
  • Amount and type of cache memory

Clock Speed

The system clock generates a series of signals. The CPU cannot preform functions faster then the clock speed. Clock speed is generally measured in gigahertz. The greater the clock speed the, the faster the instructions can be carried out.

Number of Cores

Traditional computers (Von Newman), instructions are fetched and executed one at a time. New computers have multiple cores and are able to process twice the information at the same time.

Amount and Type of Cache Memory

This is very fast memory inside the CPU. If instructions are repeated they can be stored on the cache so it is quicker to access. As cache fills up unused instructions are moved to the main memory.

Pipelining

Pipelining is a used to improve performance. At some point during the fetch-decode execute cycle at least one component is idle.

Pipelining allows for the next instructions to be fetched at the same time a process is being preformed.

Processor pipeline is divided into instruction pipeline and an arithmetic pipeline:

Instruction Pipeline

It is when instructions are moved through the processor including the fetch and execute

Arithmetic Pipeline

Represents the parts of an arithmetic operation that can be broken down and overlapped

Words and Word Sizes

Address Bus

When the processor wants to read a word of data from memory, it first puts the address of the desired word on the address bus, it first puts the address of the desired word on the address bus.

The width of the address bus determines the maximum possible memory capacity of the system.

Data Bus

The data bus transmits the data held in a word of memory, between processor components and memory.

If the data bus is 16 bits wide, a word cannot hold an integer larger than 2^16 -1. A wider bus can transmit more data at a time

How this related to Assembly Language

In assembly language, operation code (opcode) will be expressed as a mnemonic such as ADD, SUB, LDA. With 6 bits for opcode there cannot be more then 2^6 different instructions.

Different Architectures

The von Neumann architecture specifies the basic components of a computer and processor is which a shared memory and bus is used for both data and instructions.

A stored program concept can be defined as: machine code instructions are fetched and executed serially by a processor that preforms arithmetic and logical operations

  • A program must be in the main memory to be executed
  • The machine code instructions are fetched from memory one at a time.

Most computers are now built upon this principle.

Von Neumann Architecture

The same data bus is used to transfer both data and instructions. Similarly, a single bus is used to transfer the addresses of data and instructions. The same word length is used for all memory.

Harvard Architecture

Harvard architecture is used mainly for embedded Digital Signal Processing systems, such as audio and radar processing.

  • Instruction address bus can be wider then the data bus
  • There is much more instruction memory than data memory
  • Instructions may be held in a read only memory while a data memory requires read-write memory

Embedded systems include special-purpose computers built into devices often operating in real time. Harvard architecture is generally faster because data and instructions can be fetched in parallel.

Comparison of Von Neumann and Harvard Architectures

Von Neumann Harvard Architecture used in conventional processors in PC's only with control functions

  • Used in digital processing and embedded systems
  • Data and programs share the same memory
  • Instructions and data held in separate memories
  • One bus for data and instructions
  • Parallel data and instruction buses may be used
  • Programs can be optimised for size
  • Programs tend to be large

Contemporary Processor Architecture

Modern CPU chips incorporate aspect of both Von Neumann and Harvard architecture. There is one main memory, but CPU cache memory is divided into an instruction cache and data cache.
