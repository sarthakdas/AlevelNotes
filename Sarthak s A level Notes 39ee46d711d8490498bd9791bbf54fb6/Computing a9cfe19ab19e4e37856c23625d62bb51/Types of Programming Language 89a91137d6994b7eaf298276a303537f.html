Types of Programming Language

Programming Paradigms

It is a style of computer programming. Each programming language supports one of these paradigms, and some support more then one.

Procedural

Supported by Python and Pascal. This has a series of instructions that tell the computer what to do with the input in order to solve the problem. Structured Programming is a type of procedural which uses sequences, selection, iteration and recursion.

Object-orientated

Supported by Java and Python. It was developed to make it possible to abstract details of implementations away from the user, making the code reusable and easier to maintain. It is taking over from procedural programming.

Declarative

Supported by SQL. Statements have to be written to describe the problem to be solved. The language implementation decides the best way to solve it.

Functional

Supported by Haskell and Python. Functions are used as the building blocks of the program. Statements are written as a series of functions that accept input data as arguments and return and output.

Procedural Languages

Procedural languages has built in data types such as integers, reals and floats. It typically has data structures such as arrays and records.

Programmers can create their own data types such as queue, stack and hash table. This can be implemented in different ways such as using built in list data structure. All the users really need to know is the state and behaviour of the data structure.

Assembly Language

Machine code was the first language used to enter programs. The next advance in programming was using mnemonics instead if binary codes, this is called assembly code.

Each assembly language instruction translates into one machine code instruction.

Assembly code uses mnemonics to represent the operation codes and addresses, typically 2 to 4 character mnemonics. The assembler then translates the assembly language into machine code for execution.

Below is the instruction set for the Little Man Computer:

Numeric Code 1xx

Subtract - SUB

Subtract the contents of the memory address from the accumulator

Branch - use the address given as the address of the next instruction

Branch to the given address given if the accumulator is zero

Branch Instructions

The flow of a program can be altered using a conditional or unconditional branch instruction.

The conditional branch instructions BRP (Branch if positive) and BRZ(Branch if zero) branch to a given label depending on what is held in the accumulator.

The unconditional branch instruction BRA will branch what ever is held in the accumulator

Examples

Input three numbers x,y,z. Calculate and output the value of x + y - z:

INP       %Input Y into accumulator
STA Y     %Store Y 
INP       %Input X into accumulator 
STA Z     %Store X 
INP       
ADD Y 
SUB Z 
OUT 
HLT

Addressing Modes

The operation code (opcode) consists of binary digits representing the operation and a 2 digit representing the addressing mode.

There are four types of addressing mode

  • Immediate Addressing - the operand is the actual value to be operated on
  • Direct Addressing - operand holds the memory address of the value to be operated on
  • Indirect Addressing - the operand is the location which holds the address of the data
  • Indexed Addressing - the address is obtained by adding the contents of a general register a constant value

Examples

Accumulator ACC holds 25

Index register holds 6

Register 0 (R0) contains 0

LOAD immediate 4%Will put the value 4 into accumulator
LOAD direct    4%Will put 8 (contents of location 4)
LOAD indirect  4%Will put 27 (contents of the address held in location 4)
LOAD indexed  R0%Will put 15 (contents of location (index register + contents of R0))

Object-Oriented Languages

This is when we define a class as the description of what the data looks like (the state) and what the data can do (the behaviour). The user of a class sees only the state and behaviour of a data item. Data items are called objects, where an object is an instance of a class.

In a hospital system, objects might be a patient, ward, doctor, nurse and so on. Each of these objects can be defined as a class with its own set of behaviours.

General advantages of coding in Object-Oriented is:

  • Reusability objects that are coded and tested can be used again in other programs
  • OOP provides good framework for code libraries which can be changed
  • It forces designers to plan extensively, creating fewer weaknesses
  • OOP programs are easier to maintain due to its modular structure

Inheritance

Each object in its class can have sub-classes and sub-sub-classes. Suppose an program used by a estate agent defines a class called Property. Property has attributes including address, owner, type,price.

The class Properties has two sub-classes called Property_For_Rent and Property_For_Sale. The two subclasses have the same attributes as the superclass Property and in addition each has some attributes of its own.

Polymorphism

This refers to a programming languages ability to process objects differently depending on their class.

A class of objects has behaviours or methods, all of which can be inherited by its sub-classes. Sometimes some sub-classes methods might have to altered slightly from their parents class.

Polymorphism is when a subclass can redefine a method defined in its parent or super class.

Encaptulation

Certain attributes are defined in the super class. These attributes cannot be accessed directly if they are declared private; they can only be accessed inside the class methods.
