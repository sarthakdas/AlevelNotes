Operating Systems

An Operating Systems is a program or set of programs that manages the operations of the computer for the user

It acts like the bridge between the user and the computer's hardware, since the user cannot communicate with hardware directly.

The operating systems is held in permanent storage, e.g a hard-disk. A small program called the loader is held in the ROM. The loader in the ROM sends instructions to load the operating system by copying it from storage into RAM.

Functions of an OS

Apart from the UI, the operating system is able to do:

  • Memory Management
  • Interrupt Service Routines
  • Processor Scheduling
  • Backing Store management
  • Management of all Inputs and Outputs

Memory Management

When a user is running multiple applications at once. The computer must allocate a specific area of memory whilst the computer is running. So when the user moves to another application, each applications memory must be stored simultaneously. The allocation of memory is controlled by the operating system.

Paging and Segmentation

These are techniques are used to optimise memory.

  • Paging

    Memory is divided into fixed sizes of 4kb, and each program may hold more than one page. When a program which used 15kb of memory these logical memory(the pages) locations may be physically stored in for separate locations. A Page Table is used to remember where each page is being held.

  • Segmentation

    Is the logical division of address space into varying length segments which depend on the program structure.

Virtual Memory

Memory has a limit, so as more and more jobs are loaded into memory some of the data is moved off the RAM and onto the secondary storage device. The inactive data will be moved to the secondary device which starts acting like RAM.

Interrupts

An Interrupt is a signal from a software program, hardware device or internal clock to the CPU. This happens when a software terminates or requests certain services from the operating system.

They can also be triggered by a timer, that does certain mount of each task for a certain amount of time. Because of this OS's can multitask.

When the CPU receives an interrupt it suspends execution of the running program or process and disables all interrupts of a lower priority, while an Interrupt Service Routine is called to deal with it.

A check for interrupts is done after the end of each clock cycle.

Processor Scheduling

The scheduler is the module responsible for making sure the processor is being used as efficiently as possible. Their objectives are:

  • Maximise throughput
  • To be fair to all users on a multi-user system
  • Provide acceptable response times
  • Ensure hardware is kept as busy as possible

These are different scheduling algorithms:

Round Robin

This is when the to-do list is sequenced as first-in first-out. Each process is given a limited amount of CPU time called a time slice. If it does not complete it in the time then the processor moves on.

First Come Fist Served

Jobs are processed in the order in which they arrive, with no system of priorities

Shortest Remaining Time

The process with the smallest estimated time to completion is run nest. This is used to reduce the number of waiting jobs.

Multi-level feedback Queues

It gives preference to:

  • Short jobs
  • I/O bound processes
  • Separate processes into categories based on need

This allows it to move between several queues to keep it the quickest

Backing Store Management

When files and applications are loaded, they are transferred from backing storage into memory. The OS needs to keep a directory of where the files are stored so they can be accessed quickly.

Peripheral Management

This controls I/O devices. Data is transferred to a buffer to compensate for the different speeds that the peripheral and CPU operate at.

Types of Operating Systems

Distributed Operating System

A distributed operating system is a form of parallel processing which spreads the load over many computer servers.

A single job is split up into several tasks. It appears to the user to be a single system. The system is configured to share memory and tasks. Linux, Unix, and Windows are available in distributed versions.

Embedded Operating System

Embedded systems are mainly used in electronic hardware. They generally so not have a permanent storage data.

They have minimal user interface and often accept input from sensors.

Multi-tasking Operating System

This can run on a standalone computer. Most operating systems are able to run multiple jobs at once making them multi-tasking operating system.

The operating system keeps switching between jobs to make it look like all jobs are running at the same time.

Multi-user Operating System

Time-sharing operating systems are multi-user systems. A single powerful mainframe or supercomputer is connected to multiple terminals using the mainframes CPU. It dedicates time to each processor using a scheduling system.

Real-Time Operating System

Real-time operating system must be able to

  • respond quickly to inputs
  • deal with multiple inputs quickly
  • fail safe mechanism if hardware fails
  • incorporate redundancy if one component fails

BIOS (Basic Input Output System)

The BIOS is stored in the EPROM the memory that gets the computer started. The use of BIOS is to initialise and test the system hardware.

Device Drivers

A device driver is a computer program that provides a software interface to a particular hardware. This enables the OS to access the hardware functions without needing to know the details of the hardware.

Drivers are hardware and OS specific. A driver typically communicates with the device using a network bus.

Virtual Machines

A virtual machine can be defined as any instance where software is used to take in the function of the machine. including executing intermediate code or running an operating system within another.
